# Family Budget

> **Веб-приложение для управления семейным бюджетом**  
> Стек: Spring Boot 3.2.4 | Java 21 | SQL Database (PostgreSQL/MySQL) | Thymeleaf | Bootstrap 5

---

##  Описание

Система позволяет вести учет доходов и расходов, планировать бюджет, ставить финансовые цели и анализировать траты с помощью интерактивных графиков. Приложение поддерживает многопользовательский режим с разграничением прав доступа и работает на основе надежной реляционной базы данных.

---

##  Ключевые возможности

### Управление финансами
*   **Учет операций:** Добавление, редактирование и просмотр доходов/расходов с детализацией по категориям.
*   **Ролевая модель:** 
    *   **Глава семьи:** Полный доступ ко всем данным, управление бюджетом и пользователями.
    *   **Участник:** Управление своими личными тратами и просмотр общей статистики.

### Планирование и цели
*   **Бюджет:** Установка месячных лимитов с визуальным контролем исполнения (прогресс-бар с цветовой индикацией).
*   **Цели накопления:** Создание финансовых целей (семейных и личных) с отслеживанием прогресса заполнения.

### Аналитика и отчетность
*   **Графики:** Визуализация структуры расходов по категориям (библиотека ApexCharts).
*   **Фильтрация:** Гибкий выбор периода отчетности (2025–2032 гг.), фильтрация по категориям и членам семьи.
*   **Экспорт данных:** 
    *   Генерация подробных отчетов в формате **Excel (.xlsx)** (листы: Доходы, Расходы, Сводка).
    *   Резервное копирование всей базы данных в формат **JSON**.

### Интерфейс
*   **Адаптивность:** Полная поддержка мобильных устройств и десктопов (Bootstrap 5).
*   **Темы:** Переключение между светлой и темной темой с сохранением предпочтений.
*   **UX:** Плавная анимация чисел (CountUp.js) и всплывающие уведомления (SweetAlert2).

##  Быстрый старт

### Требования
*   JDK 21 или выше
*   Maven 3.8+
*   Установленный и запущенный **SQL сервер** (PostgreSQL, MySQL и т.д.)

### Настройка базы данных

1.  Создайте пустую базу данных на вашем SQL сервере:
    ```sql
    CREATE DATABASE family_budget;
    ```

2.  Настройте подключение в файле `src/main/resources/application.properties`:
    ```properties
    # Пример для PostgreSQL
    spring.datasource.url=jdbc:postgresql://localhost:5432/family_budget
    spring.datasource.username=your_username
    spring.datasource.password=your_password
    spring.jpa.hibernate.ddl-auto=update
    spring.sql.init.mode=never
    
    # Пример для MySQL
    # spring.datasource.url=jdbc:mysql://localhost:3306/family_budget
    ```
    *(Не забудьте добавить драйвер вашей БД в `pom.xml`, если он еще не добавлен)*

### Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
    cd family-budget-app
    ```

2.  **Соберите проект:**
    ```bash
    mvn clean install
    ```

3.  **Запустите приложение:**
    ```bash
    mvn spring-boot:run
    ```
    *Или используйте кнопку `Run` в вашей IDE (NetBeans/IntelliJ).*

4.  **Откройте в браузере:**
    ```text
    http://localhost:8080/login
    ```

##  Тестовые данные

При первом запуске (если включена инициализация) или после ручного импорта скрипта создаются следующие пользователи:

| Роль | Логин | Пароль | Описание |
| :--- | :--- | :--- | :--- |
| **Глава семьи** | `admin` | `admin` | Полный доступ, управление бюджетом |
| **Участник** | `mom` | `mom` | Добавление расходов, общая статистика |
| **Участник** | `son` | `son` | Добавление личных расходов |
| **Участник** | `daughter` | `admin` | Добавление личных расходов |

*(Для загрузки тестовых данных убедитесь, что файл `data.sql` находится в папке `resources` и параметр `spring.sql.init.mode` установлен в `always` при первом запуске).*

##  REST API

Приложение предоставляет endpoints для интеграции (возвращают JSON):

| Метод | Endpoint | Описание |
| :--- | :--- | :--- |
| `GET` | `/api/v1/transactions` | Получить все транзакции |
| `POST` | `/api/v1/transactions` | Создать новую транзакцию |
| `GET` | `/api/v1/incomes` | Получить все доходы |
| `GET` | `/api/v1/goals` | Получить цели накопления |
| `GET` | `/reports/excel` | Скачать Excel-отчет |
| `GET` | `/reports/backup` | Скачать JSON-бэкап базы |
